Visión General

El módulo Analítica Spreadsheet es una hoja de cálculo interactiva integrada en Generador de Reportes X que permite construir informes contables analíticos usando fórmulas personalizadas conectadas en tiempo real con la BD del ERP OFIMA.

Replica en web las funcionalidades del complemento ofima.xla de Excel: el usuario escribe =NOMBRECTA("11"), =SALDOINICIAL("11") o =SALDOFINAL("11") en cualquier celda, exactamente igual que en Excel con el add-in instalado.

Casos de uso: Balance Mensual · Estado de Resultados · Balance General Comparativo · Flujo de Caja · cualquier informe contable ad-hoc.

Integración jSpreadsheet CE v5

Cambio de API v4→v5: jspreadsheet(el, options) retorna WorksheetInstance[]. El destroy es jspreadsheet.destroy(el) (estático). La toolbar debe ir en el objeto raíz (SpreadsheetOptions), no dentro de worksheets[].

obtenerHojaActiva() — doble fallback

private obtenerHojaActiva(): any {
  // 1. Instancia almacenada localmente
  if (this.spreadsheet && typeof this.spreadsheet.getData === 'function')
    return this.spreadsheet;
  // 2. Acceso directo al elemento DOM (fallback v5)
  const hoja = this.containerRef?.nativeElement?.spreadsheet?.worksheets?.[0] ?? null;
  if (hoja && typeof hoja.getData === 'function') { this.spreadsheet = hoja; return hoja; }
  return null;
}

Carga de contexto (botón Calcular)

  1. Escanea columna A con getData(), filtra valores /^\d+$/ para obtener lista de cuentas únicas
  2. Hace POST a /api/Analitica/CargarContexto con empresa, años, meses y lista de cuentas
  3. Recibe ContextoDatos y llama OfimaFormulasPlugin.setContexto(contexto)
  4. Fuerza recálculo: itera todas las celdas y hace setValueFromCoords en cada celda con fórmula

CargarContextoAsync — ejecución paralela

Por cada cuenta en request.Cuentas lanza simultáneamente 4 queries (nombre, saldo inicial, saldo final, débito, crédito) usando Task.WhenAll:

var tareas = request.Cuentas.Select(async cuenta => {
    var nombre       = await NombreCuentaAsync(cuenta, request.Empresa);
    var saldoInicial = await SaldoCuentaAsync(cuenta, request.MesInicial, request.Acumulado, request.Empresa, request.Anio2);
    var saldoFinal   = await SaldoCuentaAsync(cuenta, request.MesFinal,   request.Acumulado, request.Empresa, request.Anio1);
    var debito       = await SaldoDBCRAsync(cuenta, request.MesFinal, "DB", request.Empresa, request.Anio1);
    var credito      = await SaldoDBCRAsync(cuenta, request.MesFinal, "CR", request.Empresa, request.Anio1);
    return new { cuenta, nombre, saldoInicial, saldoFinal, debito, credito };
});
var resultados = await Task.WhenAll(tareas);

SaldoCuentaAsync — acumulado vs período

if (acumulado == "A")
    // Suma desde mes 1 hasta @periodo  → AND Month(FechaMovimiento) <= @periodo
else
    // Solo el mes exacto              → AND Month(FechaMovimiento) = @periodo

SaldoDBCRAsync — naturaleza DB/CR

  WHEN @naturaleza = 'CR' THEN m.Abono

Cargar Plantilla — Restauración de Formato

cargarPlantilla(plantilla: PlantillaAnalitica): void {
  const parsed = JSON.parse(plantilla.contenido);

  // Restaura parámetros del panel de consulta
  if (parsed.params) this.params = { ...this.params, ...parsed.params };

  // Re-inicializa jSpreadsheet con datos + formato completo
  this.inicializarSpreadsheet(parsed.data, {
    style:      parsed.style,       // pasado a WorksheetOptions.style
    mergeCells: parsed.mergeCells,  // pasado a WorksheetOptions.mergeCells
    colWidths:  parsed.colWidths,   // aplicado sobre columns[i].width
    rowHeights: parsed.rowHeights,  // reservado
  });
}

Flujo Completo de Datos — Calcular

  1. Usuario configura Empresa, Año 1/2, Mes Inicial/Final, Acumulado en el panel de parámetros.
  2. Usuario hace clic en Calcular.
  3. El componente escanea la columna A del grid con getData() y extrae códigos numéricos únicos.
  4. POST /api/Analitica/CargarContexto envía la lista de cuentas al backend.
  5. El repositorio lanza queries SQL en paralelo (Task.WhenAll) por cada cuenta.
  6. El backend retorna ContextoDatosDto con diccionarios de nombres, saldos, débitos y créditos.
  7. El componente llama OfimaFormulasPlugin.setContexto(contexto).
  8. Fuerza recálculo del grid: itera todas las celdas y re-escribe las fórmulas con setValueFromCoords().
  9. jSpreadsheet re-evalúa cada fórmula → el plugin retorna el valor del contexto → la celda muestra el resultado.

Conclusión para este proyecto

jspreadsheet-ce v5 (MIT) cubre el 100% de los requerimientos del módulo Analítica Spreadsheet: grid editable, toolbar nativa, múltiples hojas (v5), estilos por celda, merges, fórmulas custom OFIMA, exportar CSV, plantillas con formato completo — todo sin costo de licencia.

La única funcionalidad ausente relevante para una mejora futura sería Exportar a XLSX real (actualmente solo CSV) y Importar desde Excel, ambas disponibles como extensiones Pro.
